copy (sourceTable, copyTables, copyMetatables)
Returns a shallow or deep copy of the specified table.

Parameters

  • sourceTable: The table to copy.
  • copyTables: False if the specified table and its resulting copy should share references to child elements that are also tables; True if child tables should be copied recursively. Defaults to false.
  • copyMetatables: False if the specified table and its child table elements should share metatables with the resulting copies of the tables; True if metatables should be copied recursively. Defaults to false.

Usage

  • table2 = copy( table1 )
  • table2 = copy( table1, true )
  • table2 = copy( table1, true, true )
  • table2 = copy( nil ) -- nil
  • table2 = copy( "non-table" ) -- "non-table"

Return value:

A copy of the specified table.

See also:

  • http://lua-users.org/wiki/CopyTable
getNumericKeysSorted (target)
Returns a new table consisting of the sorted numeric key values of the specified table. Useful for iterating through a mixed table that contains associative array entries in addition to traditional numeric array entries.

Parameters

  • target: The target table to search for numeric key values.

Usage

  • getNumericKeysSorted( { [ 4 ] = "a", [ 1 ] = "b", hi = "bye", [ 2 ] = "c" } ) -- { 1, 2, 4 }
  • getNumericKeysSorted( nil ) -- nil
  • for index, value in ipairs( getNumericKeysSorted ( mixedTable ) ) do

Return value:

The sorted numeric key values of the specified table.
isNonEmptyTable (target)
Indicates whether the specified parameter is a table that contains at least one element.

Parameters

  • target: The target parameter to evaluate.

Usage

  • isNonEmptyTable( nil ) -- false
  • isNonEmptyTable( "Hi" ) -- false
  • isNonEmptyTable( { } ) -- false
  • isNonEmptyTable( { "" } ) -- true

Return value:

True if the specified parameter is a table that contains at least one element; otherwise, false.
pack (...)
Returns a table that contains all of the passed parameters.

  • ...: A list of optional parameters with which to populate the returned table.

Usage

  • pack ( 1, 2, 3 ) -- { 1, 2, 3 }
  • pack ( 1, nil, 2, 3 ) -- { 1, nil, 2, 3 }

Return value:

A table that contains all of the passed parameters.
selectByNestedIndex (target, ...)
Returns the specified named element of the specified table, traversing nested table elements as necessary. If the specified table is nil, or lacks the specified named element, returns nil.

Parameters

  • target: The target table to search for the specified named element.
  • ...: A list of element names, with each name representing a step deeper into the hierarchy of nested table elements.

Usage

  • selectByNestedIndex( { hi = { bye = 9 } }, "hi", "bye" ) -- 9
  • selectByNestedIndex( { hi = { bye = 9 } }, "hi" ) -- { bye = 9 }
  • selectByNestedIndex( { hi = { bye = 9 } }, "hi", "there" ) -- nil
  • selectByNestedIndex( nil, "hi", "bye" ) -- nil

Return value:

The specified named element of the specified table, if available; otherwise, nil.

See also:

  • updateByNestedIndex
updateByNestedIndex (value, target, ...)
Updates the specified named element of the specified table to the specified value and returns it, traversing and creating nested table elements as necessary. If the specified table is nil, returns nil.

Parameters

  • value: The value to which to set the specified named element in the specified table.
  • target: The target table to search for the specified named element.
  • ...: A list of element names, with each name representing a step deeper into the hierarchy of nested table elements.

Usage

  • updateByNestedIndex( 9, { hi = { bye = 0 } }, "hi", "bye" ) -- { bye = 9 }
  • updateByNestedIndex( 9, { hi = { } }, "hi", "bye" ) -- { hi = { bye = 9 } }
  • updateByNestedIndex( { bye = 9}, { hi = { } }, "hi" ) -- { hi = { bye = 9 } }
  • updateByNestedIndex( 9, { hi = { } } ) -- { hi = { } }
  • updateByNestedIndex( 9, nil, "hi", "bye" ) -- nil

Return value:

The specified named element of the specified table, if the table is not nil; otherwise, nil.

See also:

  • selectByNestedIndex
